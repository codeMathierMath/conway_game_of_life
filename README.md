# Conway’s Game of Life (Java)

This project implements a subset of **Conway’s Game of Life** as a discrete
dynamical system on a two-dimensional grid. It was written to deepen
understanding of state representation, rule-based evolution, and boundary
conditions, rather than to serve as a fully general or optimized simulation.

Visualization is provided via `StdDraw`, but the emphasis of the project is on
the underlying model rather than graphics.

---

## Concepts Demonstrated

- State abstraction using a 2D grid (an array of arrays)
- Discrete dynamical systems
- Deterministic rule-based evolution
- Toroidal boundary conditions via modular arithmetic
- Separation of state evolution from rendering
- Procedural decomposition

---

## Overview

The world is represented as a square grid of cells, each of which is either
**alive** or **dead**. At each time step, the state of every cell is updated
simultaneously according to Conway’s four canonical rules.

Boundary conditions are handled by treating the grid as a torus (wrapping at
edges). This choice simplifies the evolution logic and avoids special-case
edge handling.

Initial configurations are generated by placing multiple copies of known
patterns—such as **gliders**, **pulsars**, and **lightweight spaceships
(LWSS)**—at random locations.

---

## Design Philosophy

This implementation prioritizes **clarity and correctness** over abstraction
or performance optimization.

In particular:
- Neighbor states are computed explicitly rather than via compact tricks
- Pattern initialization is encoded directly for transparency
- State evolution is handled by producing a new grid each step (no in-place
  mutation)

The goal is to make the logic easy to read, reason about, and modify.

---

## Running the Program

This program requires `StdDraw.java` to be present.

Compile:
```bash
javac ConwayGameOfLife.java StdDraw.java
```
